# Money Watcher — POO

Sistema desenvolvido com foco no aprendizado de **Programação Orientada a Objetos (POO)** aplicado ao gerenciamento financeiro.

O projeto simula funcionalidades básicas de controle de gastos e receitas, utilizando conceitos fundamentais de orientação a objetos para organizar melhor a estrutura do código e separar responsabilidades entre classes.

---

# Objetivo

Este projeto foi criado com o objetivo de praticar:

* Estruturação de projetos utilizando POO
* Organização e modularização de código
* Separação de responsabilidades
* Reutilização de código
* Modelagem de entidades do mundo real

---

# Conceitos de POO Utilizados

## Encapsulamento

Os dados das classes são protegidos através de atributos privados e métodos responsáveis pelo acesso e alteração dessas informações.

### Aplicações no projeto

* Controle de saldo
* Manipulação de transações
* Proteção de dados financeiros

### Aprendizados

* Maior segurança dos dados
* Melhor controle sobre alterações
* Organização mais limpa do sistema

---

## Abstração

As classes representam entidades reais do sistema financeiro de forma simplificada.

### Exemplos

* Conta
* Usuário
* Transação
* Controle financeiro

Cada classe contém apenas os atributos e comportamentos necessários para sua função.

### Aprendizados

* Melhor organização lógica
* Facilidade de manutenção
* Separação clara de responsabilidades

---

## Herança

A herança foi utilizada para reutilizar características comuns entre diferentes classes.

### Aplicações no projeto

* Compartilhamento de propriedades
* Especialização de tipos de operações financeiras

### Aprendizados

* Redução de código repetido
* Reutilização de estruturas
* Melhor manutenção do projeto

---

## Polimorfismo

O polimorfismo permitiu trabalhar com métodos semelhantes possuindo comportamentos diferentes dependendo da classe utilizada.

### Aprendizados

* Maior flexibilidade
* Código mais modular
* Facilidade para expansão futura
